1 MH_Widgets
2 =========
3 
4 This sets of classes will provide a quick way to create widgets in PHP and integrate thow widget in your HTML.
5 
6 The workflow of a proper widget is:
7 
8 1. Create an instance of MH_Widget
9 2. Call the instance as a function (to trigger the magic `__invkoke`) passing three arguments (two optional): path, config array, className
10 3. under the hood the class will:
11  - check if the class is already loaded using the className parameter or the last bit of the path
12  - if there is no instance include the php file in the provided path but inside the Widgets folder
13  - create a new instance of the requested widget and pass the config object
14  - call the instance as function (to trigger the magic `__invkoke`) and return the result
15 
16 The single widget workflow is:
17 
18 1. from the magic invoke the the `controller` method is called
19 2. the `controller` method call the `model` method to get data
20 3. the `controlle`r will call and return the view method passing the data from the model
21  - if the `prevent_view` kay is found in the config array the controller will return the data from the model without call the view
22 
23 Edge Cases
24 ---
25 
26 To be a proper widget the class must exted the MH_WIdgetBase class and override the `__invoke`, `controller`, `model` and `view` methods, if a non poper widget is passed to the MH_Widget the function will return an instance of the called class without trigger the whole process
27 
28 There are two main way to use this set of classes:
29 1. Use MH_Widget class out of the box
30 2. User the integrated parser to use just simple HTML markup and have the rendered result
31 
32 Out of the box
33 ---
34 
35 Simple follow the previus workflow or use this snippet:
36 
37 ```php
38 require_once "Class/MH_Widget.php";
39 
40 $widget = new \MH_Widget\MH_widget();
41 
42 echo $widget("Gravatar", ["email" => "youremail@gravatr.com", "size" => 80]);
43 
44 ```

46 Parser
47 ---
48 
49 With the integrated parser you cna use a custom html tag MHwidget with attributes to render content from the widget, and example of a working tag:
50 ```html
51 <MHwidget size="128" email="youremail@gravatr.com">Gravatar</widget>
52 ```
53 
54 the very same result can be use the config attribute, the content will be trated as serialized array sp ypu can use
55 ```html
56 <mhwidget config='a:2:{s:4:"size";s:3:"128";s:5:"email";s:21:"youremail@gravatr.com";}'>Gravatar</mhwidget>
57 ```
